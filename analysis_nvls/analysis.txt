[Key Themes]
- IA core hybridization -- compile-time stub (RTL Core PM entries): the fundamental HSLE compile-time change is core_client.v -- a stub that replaces cdie_p1278_core with empty par_mlc/icore while keeping par_pm as live RTL. Five supporting files cascade from this: cdie_emu_tlms.sv, cdie_pwr_jem_cstate_tracker.sv, core_top_ti.v, and txte_tlm_top_collectors_binds.vs each remove bind statements pointing into the absent hierarchy; verif/emu/buildit/EmuGen.py removes the gen-stage probe generation for those same modules.
- par_pm preservation is explicit: cdie_hsle.py forces par_pm.gp_side_pok and par_pm.pm_side_pok on every Big Core and Atom Core path (sfc_bcslice0.par_bsX.coreX_wrap.coreX). par_pm_fwc.v provides companion force-write collateral for PM state-machine inputs otherwise driven by the missing par_mlc/icore.
- Hybrid mux bridges IOSF SB from RTL to Simics: cdie_emu_tb.sv instantiates cdie_emu_hybrid_mux_xtor -- the per-core RTL mux that routes IOSF SB traffic either from the real Simics IA model or from the HSLE transactor path. hyb_sel_iosf_sb is forced in cdie_hsle.py for every hybridized core, directing SB traffic through the Simics path away from stubbed RTL endpoints.
- IA core removal in this NVL-S diff: the clearest behavioral change is that src/val/emu/testbench/py_lib/iosf_sb_xtors_configuration_workaround.simics now wraps the old manual xtor creation inside `if ((cmd_line_opt->hsle) == FALSE)`. HSLE stops using the legacy per-core workaround path; the removed hookup is the manual workaround layer, not the IOSF SB protocol itself.
- How IOSF SB remains intact: the replacement path is Simics/hybrid-core based. hybrid_core_init.py rewires sideband preservation through hybrid-core smart mapping: mb.pcd.pci_mem, mb.pcd.p2sb0.bank.sbreg_bank, redirect regs at 0x41200f90d0/d4/d8, and a sai_change_funny_io callback. verif/emu/transactors.json keeps the iosf_sb_xtor module active and updates the hybrid-core/transactor packages.
- reglist/common/emu/nvlsi7_p4e8_pythonsv_iosf_sb_xtors_zse.list: unchanged between SLE and HSLE. Explicitly enables die-specific xtors with +options PCH_IOSF_SB_XTORS=1 PCD_IOSF_SB_XTORS=1 CDIE1_IOSF_SB_XTORS=1 HUBDIE_IOSF_SB_XTORS=1. Validation intent for IOSF SB preserved.
- Integration hotfix IOSF SBC endpoints: all checked integration/hotfix/rtl/*/sip/*/subIP/iosf_sbc_ep/ directories are byte-for-byte identical between SLE and HSLE. IOSF SB preservation is not from RTL endpoint edits; it is preserved above RTL by keeping collateral/tests and changing the hybrid-core/transactor hookup.
- Files not yet in either snapshot (developer-noted): core_client.v, par_pm_fwc.v, cdie_emu_tlms.sv, cdie_pwr_jem_cstate_tracker.sv, core_top_ti.v, txte_tlm_top_collectors_binds.vs, and EmuGen.py probe-exclusion changes must be created/patched when converting a clean SLE model. The converter cannot auto-apply these from the current donor snapshot.
